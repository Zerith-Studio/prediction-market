This page records the implementation maturity of each component: **implemented and
tested**, **implemented but untested**, **off-chain by design**, and **designed but not
implemented**.

## On-chain — implemented and tested

The full binary settlement path is implemented and deployed on Solana devnet at program
ID `3fdgRPcZnwWcaGi197dkZDyq24VHoWJcGzKTVfMxNPWs`:

- Signed order → engine match → `settle_match` — on-chain ed25519 verification, per-order
  fill accounting, and NORMAL/MINT/MERGE money movement.
- `resolve_market` → `redeem` — moving USDC 1:1 against escrow.

All eight program lifecycle tests pass on a local validator, covering the three settle
paths, cancel-fails-closed, resolution, and redemption. The same lifecycle runs end to
end against devnet with every balance asserted.

## Off-chain — implemented and tested

The Go backend — matching, store, exchange, crank builder, REST + WebSocket API, RFQ,
precision, MM bot, and lifecycle — passes an HTTP end-to-end suite against Postgres,
including adversarial coverage: 28 attack vectors returning clean 4xx, concurrent load
with zero 5xx, and a fixed uint64-overflow case.

## Off-chain by design

- **Precision pools** are pool-based (ADR 0006). There is no priced complete set to
  escrow, so they settle off-chain.
- **Combos** currently run off-chain behind an interface seam (ADR 0007); the designed
  on-chain path is listed below.

## Implemented but untested

- **Mobile UI screens** — the client library layer runs end to end against a live
  backend, but the screens have no device-level verification yet.
- **Frontend `randomSalt()`** — the salt-rounding fix applied in the mobile encoder is
  still pending in `frontend/lib/borsh.ts` (see [Signed messages](/onchain/signed-messages)).

## Designed but not implemented

- On-chain `combo_accept` / `resolve_combo` — typed stubs returning `NotImplemented`
  (ADR 0004).
- Oracle tiers **(b)** commit-reveal / challenge window and **(d)** provider-signed data
  verified on-chain (ADR 0005). Tier **(a)**, the single resolver key, is implemented.
- The productized combo void rule (voided leg = push) — specified, not yet built.
