## Proven on devnet (the floor)

The never-cut hill of ADR 0007 is done, reproducible via `go run ./cmd/devnet-e2e` with
every balance asserted:

- Program deployed at the pinned ID (2026-07-15).
- Signed order → engine match → **Go crank `settle_match` on devnet** (ed25519 verify,
  fill accounting, MINT path).
- `resolve_market` → `redeem` moving USDC 1:1.

All eight program lifecycle tests (`npm test`) pass on a local validator, covering all
three settle paths, cancel-fails-closed, resolution, and redemption.

## Verified off-chain

The entire Go backend (matching, store, exchange, crank builder, API, WS, RFQ,
precision, bot, lifecycle) passes an HTTP end-to-end suite against real Postgres,
including adversarial tests (28 attack vectors all clean 4xx, concurrent load with zero
5xx, a fixed uint64-overflow exploit). The web frontend passed a scripted browser e2e
against the live stack.

## Written but unverified

- **Mobile UI screens** — the library layer ran green end-to-end against a live server,
  but no physical-device click-through has happened; every screen is 🟡 until a human
  does that pass. Privy login on mobile is blocked on a dashboard allowlist setting.
- **Privy path on mobile** — never exercised against a real Privy app.
- **Frontend `randomSalt()`** — the salt-rounding bug fixed in `mobile/` is flagged as
  still present in `frontend/lib/borsh.ts` (see `progress.md` 2026-07-16).
