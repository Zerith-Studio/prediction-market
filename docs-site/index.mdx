PitchMarket is a football prediction exchange built for the Superteam × TxODDS hackathon
("Prediction Markets & Settlement" track). Users trade binary outcome markets on live
World Cup fixtures priced from the TxLINE data feed. Matching is off-chain for speed;
money moves on-chain, non-custodially, per cryptographically signed orders.

## The one-paragraph pitch

Your funds are safe during trading because the settlement contract moves them only per an
order **you** signed, at a price at least as good as your limit. The operator matches
orders and pays gas, but it can never forge a trade, over-fill your order, or take
custody. Matching is off-chain (snappy, no wallet popup per trade); settlement, resolution,
and redemption are on-chain on Solana devnet.

## Products

| Product | What it is | Settlement |
|---|---|---|
| **Binary markets** | YES/NO outcome markets per fixture template (e.g. draw-no-bet, over/under) | Fully on-chain: signed order → operator crank → `settle_match` → `resolve_market` → `redeem` |
| **Combos (RFQ)** | Parlays: taker holds "all N legs win", an MM quotes the other side | Off-chain for the demo, behind an interface seam; the on-chain `combo_accept`/`resolve_combo` path is designed (ADR 0004) but stubbed |
| **Precision pools** | Guess a match stat exactly; pool-based, winners paid by losers, σ-normalized closeness score | Off-chain (Tier 2 by design, ADR 0006) |

## Current status

The trustless floor is **done and proven on devnet**: the program is deployed at the
pinned ID `3fdgRPcZnwWcaGi197dkZDyq24VHoWJcGzKTVfMxNPWs`, and a full lifecycle — signed
order → engine match → Go crank `settle_match` (v0 transaction + address lookup table) →
`resolve_market` → 1:1 `redeem` — has run green with balance assertions
(`go run ./cmd/devnet-e2e`).

Live status, verified-vs-unverified breakdown, and the changelog live in
[`progress.md`](https://github.com/Zerith-Studio/prediction-market/blob/main/progress.md)
at the repo root — that file is the single source of truth for "what actually works
today". See also [Status & verification](/reference/status).
